Motivation

Betreiber analysieren/verkaufen private Nutzerdaten.
Beispiele:

  • Personalisierte Werbung setzt das Auswerten von Nutzerdaten voraus
  • Google liest E-mails
  • Facebook-Apps erhalten (nach Bestätigung) Zugriff auf Profildaten
  • WhatsApp benötigt Zugriff auf Handy-Kontakte
  • Cloud-Anbieter außerhalb der EU unterliegen anderen Datenschutzrichtlinien

Motivation

Nutzer von Cloud-Diensten / Chat-Anwendungen / Sozialen Netzwerken können jederzeit ausspioniert werden

  • Staat/Polizei kann Daten einsehen
  • Kein System ist 100% sicher
  • Problem:
    Können private Nutzerdaten vom Anbieter gelesen werden, dann besteht potenziell die Möglichkeit eines Diebstahls/Abgreifens der Daten durch Dritte.
  • Lösung:
    Ende-zu-Ende Verschlüsselung

Motivation

Verschlüsselung ist unpraktikabel

  • Benutzer will nichts installieren/einrichten
  • net of trust -> Aufwändige Verfahren zur Verifizierung von "Trusted Usern"
  • Ende-zu-Ende Verschlüsselung benötigt Software (auf beiden Seiten der Kommunikation)
  • "Wer kann mir eine verschlüsselte Nachricht senden?"

Funktionsweise

Schlüssel generieren

  1. Bei der Registrierung des Benutzerkontos wird ein RSA-Schlüsselpaar auf Clientseite generiert
  2. Privater Schlüsselanteil wird mit Benutzerpasswort verschlüsselt
  3. Benutzername, Passwort und Schlüssel werden anschließend auf den Server übertragen und von diesem gespeichert

PEP Chat - Fazit

Vorteile

  • Server dient als Zertifizierungsstelle
    -> Klarnamen statt Fingerprints
  • Daten bleiben auf dem Server gespeichert und sind nur mit dem zugehörigen Benutzerpasswort zugänglich
  • Weniger Fachkenntnisse erforderlich ;)
  • Anwendung ist "Open Source"

PEP Chat - Fazit

Nachteile des PEP-Chats

  • Mehr Rechenaufwand auf Klientseite
    • Schlüsselgenerierung sehr Rechenaufwendig
    • Javascriptausführung vergleichsweise langsam
    • Server kann Nachrichten nicht lesen. Filtern von Nachrichten (auf Spam/Schimpfworte) musss auf Klientseite statt finden
  • "Verschlüsselung im Browser ist unsicher"
  • Dem Dienstanbieter/Serverbetreiber muss vertraut werden

PEP Chat - Fazit

Ausblick / Verbesserungsmöglichkeiten

  • Authentifizierung mittels Keysigning
  • Kalender basierend auf vorhandener Verschlüsselung
  • Performanceverbesserungen (Bsp.: Hybride-Verschlüsselung)
  • Gruppenchat
  • Schnittstellen zu anderen Chat-Klients (z.B. XMPP, OTR-Verschlüsselung)
